Detailed Description

Klasa jest interfejsem dla użytkownika użytkującego silnik.

Taki interfejs będzie widział użytkownik silnika z wnętrza swoich klas odpowiedzialnych za fabułę. Użytkownik musi mieć ograniczone możliwości, żeby przypadkiem nie namieszał.

Funkcje są pogrupowane tematycznie przy pomocy klas zagnieżdżonych. Wszystkie funkcje jakie powinien mieć dostępne należy deklarować własnie w klasach zagnieżdżonych. Zasadniczo klasa EngineInterface nie powinna mieć oprócz tego żadnych funkcji API.

Zmienne i składniki silnika należy deklarować w strukturze EngineContext. Dzięki temu będzie do nich dostęp z poziomu komend, ponieważ każda komenda dostaje referencję na tę strukturę.
